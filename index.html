Healthcare Digital Twin Platform

A full-stack platform that predicts chronic-disease risk and simulates a patient's health trajectory as a digital twin.

Group Project
Problem
Early risk screening for heart disease and diabetes is usually scattered across single-purpose calculators. There was no single place where a user could see a prediction, a simulation of how their health might trend, and a concrete next step — so I proposed building one.
My role
This was a group mini-project for my semester, but the idea and early direction were mine — I initiated the concept, having been interested in healthcare technology for a long time, designed the initial prototype and system architecture, and led implementation of the ML pipelines, the Flask REST API, and the digital twin simulation engine, while teammates built out the chatbot and reporting modules.
Features
  • Heart disease & diabetes prediction (Cleveland & Pima datasets)
  • Digital twin simulation engine for health trajectory
  • AI chatbot for personalised diet recommendations
  • Automated PDF health report generation
Challenges
Integrating 6+ components — two separate ML models, a simulation engine, a chatbot, and reporting — behind one consistent API, using OOP design patterns to keep the codebase maintainable across a multi-person team.
Stack
PythonFlaskScikit-learnREST APIOOP Design
Impact: Won 2nd Prize at the Kalpana 2K25 Project Expo on Engineer's Day.

AshaAI — Opportunity Copilot

An AI copilot that matches Indian citizens to the government schemes and scholarships they actually qualify for.

Shipped
Problem
Millions of eligible citizens miss out on government schemes and scholarships because eligibility criteria are scattered across departments and written in dense official language — often not even in the citizen's first language.
My role
Solo-built the entire backend and Gemini API integration — the eligibility reasoning engine, the auth and data layer, and the multilingual voice pipeline.
Features
  • Eligibility reasoning engine over profile data (education, income, state, category)
  • Direct links into the actual application portals
  • Multilingual voice input via Sarvam AI
  • Secure auth and persistent user profiles
Challenges
Turning unstructured, department-specific eligibility rules into a reliable reasoning layer, and supporting voice input across Indian languages without compromising response quality.
Stack
PythonGemini APIMongoDBFirebaseSarvam AI
Impact: Cuts through India's fragmented scheme-discovery process by connecting users directly to what they're eligible for.

Money Laundering Detection System

An ML pipeline that flags suspicious financial transactions, tuned to 85.6% accuracy against a baseline model.

Shipped
Problem
Manual transaction review doesn't scale, and simple rule-based flags miss the more nuanced patterns that indicate laundering.
My role
Solo project — built the feature engineering pipeline, tuned the model, and built the review dashboard.
Features
  • Random Forest classifier tuned against a Logistic Regression baseline
  • Feature engineering pipeline across ~2,300 transactions
  • Flask API serving predictions to a Tailwind CSS review dashboard
Stack
PythonScikit-learnFlaskTailwind CSS
Impact: 85.6% detection accuracy, a meaningful lift over the Logistic Regression baseline.
Recognition: 3rd Place (National) — presented at the C-DAC Bihar Hackathon, a national-level hackathon.
